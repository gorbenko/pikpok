# pikpok
PikPok is a modern application for watching short videos

https://gorbenko.github.io/pikpok/

В решении преследовал принцип "не усложняй", использованы базовые возможности браузера для подобных задач, `IntersectionObserver` и `scroll-snap-*`.

Несколько слов о процессе:
- не стал городить класс или прототипы
- оставил обзёрвинг для автоплея при возврате на предыдущие видео. Этого можно не делать оставив ручной плей. Либо оставлять только 3 предыдущих видео, если же пользователь умотал на -5 видео наверх то, либо детектить и запускать, либо ручной плей.
- так же можно удалять N просмотренных видео из верстки и кода
- сперва начал делать на одном теге video, из соображений производительности. Отказался по двум пунктам: 1. Нужно чтобы следующее видео проглядывалось когда чуть скролишь вниз, т.е. нам физически нужно видеть превью следующего видео. 2. Сложность реализации, пляски с хранением состояния, отматывание к предыдущим видео.
- имена классов выношу в `const`, хорошая практика позволяющая избежать кривых dom-запросов при изменении структуры верстки.
